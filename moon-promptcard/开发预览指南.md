# Moon PromptCard 本地开发与预览指南

面向在本机调试这个 Chrome 扩展的完整流程。第一次按顺序走一遍即可。

## 0. 前置

- 安装 Node.js 18+（建议 20/22）。
- 在项目根目录 `moon-promptcard/` 下执行命令。

---

## 1. 三个命令的区别

| 命令 | 作用 | 什么时候用 | 是否常驻 |
| --- | --- | --- | --- |
| `npm install` | 按 `package.json` 安装依赖到 `node_modules` | 首次，或依赖变动后 | 否，跑完退出 |
| `npm run build` | `tsc` 类型检查 + 两次 Vite 打包，产物写入 `dist/` | 正式验证、上架前 | 否，跑完退出 |
| `npm run dev` | 监听构建，但**只盯 popup / options / service-worker** | 只改弹窗或后台时 | 是，`Ctrl+C` 结束 |
| `npm run watch` | **同时监听两套构建**（含 content-script 悬浮条） | 日常开发，推荐 | 是，`Ctrl+C` 结束 |

> 为什么需要 `watch`：content-script 不能是 ES module，所以它由独立的 `vite.content.config.ts` 打成自包含 IIFE。`dev` 不监听这份配置，改悬浮条相关代码不会重建——`watch` 把两份配置一起监听。

---

## 2. dist 在哪

构建产物在：

```
moon-promptcard/dist/
├─ manifest.json
├─ popup.html / options.html
├─ service-worker.js
├─ content-script.js
├─ assets/        # 打包后的 JS / CSS
└─ icons/
```

`dist/` 需要先跑一次 `npm run build` 或 `npm run watch` 才会生成或更新。**改了源码后没重新构建，`dist/` 就还是旧的。**

---

## 3. 在 chrome://extensions 加载 dist

1. 地址栏输入 `chrome://extensions` 回车。
2. 打开右上角「开发者模式」。
3. 点「加载已解压的扩展程序」。
4. 选择 `moon-promptcard/dist` 文件夹（注意是 `dist`，不是项目根目录）。
5. 出现 Moon PromptCard 卡片即成功。点工具栏拼图图标把它「固定」到工具栏，方便点开。

---

## 4. 预览 Popup

点工具栏上固定的 Moon PromptCard 图标，弹出的就是 Popup。可以：

- 切换「内置服务 / 自定义 API」。
- 在自定义 API 下填 Base URL / API Key / 模型，点「测试本地 API」。
- 查看历史 / 刷新 / 进入设置。

调试：右键扩展图标 →「审查弹出内容」（Inspect popup），打开 DevTools 看控制台与元素。

---

## 5. 预览 content script 悬浮分析条

悬浮条注入在**普通网页**里，不在 Popup 内。

1. 打开任意带输入框的普通网站（任意聊天框、表单页，或自建一个含 `<textarea>` 的页面）。
2. 在 Popup 或设置页打开「悬浮球」开关。
3. 页面左侧出现玻璃胶囊条，打开其中的「悬浮分析」开关。
4. 在输入框写提示词，或选中一段文本。
5. 点橙色分析按钮，结果卡浮现——可拖拽、收起、关闭、复制优化结果、一键替换原文。

注意：`chrome://` 系统页、Chrome 网上应用店等受保护页面不允许注入，请用普通网站测试。
调试 content script：在测试网页上按 F12 打开 DevTools，控制台来源选 Moon PromptCard 的注入上下文。

---

## 6. 改完代码后如何刷新

保持 `npm run watch` 运行时，`dist/` 会自动重建，但 Chrome 仍需手动生效：

- **改 Popup / Options**：重新打开弹窗即可；必要时到 `chrome://extensions` 点该插件卡片的刷新（↻）。
- **改 service worker（后台）**：到 `chrome://extensions` 点刷新（↻）。
- **改 content script（悬浮条）**：点刷新（↻），**并刷新正在测试的网页**（content script 只在页面加载时注入）。

口诀：UI 改动 → 重开弹窗；后台 / 悬浮条改动 → 扩展页点 ↻（悬浮条再额外刷新网页）。

---

## 7. 推荐开发流程

```bash
npm install      # 仅首次
npm run watch    # 保持运行，自动重建 dist
# 改代码 → 按第 6 节刷新插件
```

收尾正式验证时再跑一次完整构建：

```bash
npm run build    # 含 tsc 类型检查，确认无报错
```

> 提示：若在某些受限环境下 `npm run build` 报 `EPERM ... unlink dist/...`，是旧 `dist/` 文件被占用/锁定所致。删除 `dist/` 文件夹后重跑即可（本机正常文件系统通常不会出现）。
