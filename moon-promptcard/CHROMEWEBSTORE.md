# Chrome Web Store 上架资料 · Moon PromptCard

## 插件名称
Moon PromptCard

## 简短描述（≤132 字符）
悬浮式提示词分析助手：对当前输入框或选中文本评分、诊断、一键优化，支持自定义 OpenAI 兼容 API。

## 详细描述
Moon PromptCard 是一个浏览器悬浮式提示词分析与优化工具，面向使用图像 / 视频生成模型（如 Veo、Seedance、Kling、Omni、GPT Image、Gemini Image、Midjourney、Stable Diffusion）的创作者。

在任意网页的输入框里写提示词时，点开页面左侧的悬浮分析条即可：
- 对当前输入框内容或你选中的文本进行**评分**与**问题诊断**；
- 给出**优化建议**与**优化后的完整提示词**，并附带常用**负面限制**；
- 一键**复制**优化结果，或直接**替换**原输入框内容；
- 结果卡片支持拖拽与收起，深色 / 浅色网页都清晰可读。

两种服务模式：
- **自定义 API**：填写你自己的 Base URL / API Key / 模型，直接调用任意 OpenAI 兼容接口，配置只存在你本地浏览器。
- **内置服务**：登录后由官方后端完成认证、计次、计费与模型调用（当前版本为占位，逐步开放）。

分析维度涵盖：主体是否明确、场景与动作是否清楚、镜头语言是否完整、风格是否统一、画幅 / 时长 / 比例是否明确、是否需要固定角色或产品、是否存在冲突词、是否缺少负面限制、是否可直接生成等。

## 权限说明
- `storage`：保存语言、主题、服务模式、自定义 API 配置与最近一次分析结果到本地。
- `activeTab` + `scripting`：在当前标签页读取你主动选择的文本或聚焦输入框内容，并把优化结果写回。
- `contextMenus`：提供「用 Moon PromptCard 分析选中文本」右键菜单。
- `host_permissions`：未单独申请；通过 `content_scripts.matches: <all_urls>` 注入悬浮分析条。

### 为什么使用 `<all_urls>`
提示词输入框可能出现在任意网站（各类创作平台、文档、聊天页）。为让用户在任何页面都能调出悬浮分析条，content script 需匹配 `<all_urls>`。插件**默认不读取任何内容**，仅在用户点击分析按钮时，才读取当前选中文本或聚焦输入框内容，绝不自动采集整页 DOM。

## 隐私说明
见 `PRIVACY.md`。要点：
- 仅在用户主动操作时读取输入框 / 选中文本；
- 不自动收集整页内容；
- 自定义 API Key 只存本地，不上传官方服务器；
- 内置服务模式下，被分析的提示词文本会发送到官方后端用于生成分析结果。

## 是否有内购
有（内置服务的分析次数 / 套餐），通过外部支付页（Stripe / Paddle / Lemon Squeezy 或自建）完成，不在插件内处理支付。

## 是否使用远程代码
否。所有 JavaScript 均随扩展打包，运行时不下载或执行任何远程脚本，不使用 `eval`。

## 数据使用说明
- 收集的数据：用户主动提交分析的提示词文本（仅内置服务模式会发送至官方后端）。
- 不收集：浏览历史、整页内容、键盘记录、个人身份信息。
- 自定义 API 模式下，提示词仅发送到用户自己配置的接口地址。
- 不向第三方出售或转移用户数据。
